# ReadWeave（织读）评测工具与公开语料计划

> 日期：2026-07-14
> 目的：用可重复证据验证上下文“尽可能全、同时尽可能少”，并持续运行端到端回归。

## 1. 评测原则

1. 公开语料覆盖多结构、多语言、多长度和多术语密度，不只测试一种文章。
2. 固定原文快照、问题、期望证据段落和可接受答案要点，避免来源更新后静默改变结果。
3. 每次运行记录源版本、工作流版本、模型配置、提示词哈希、上下文标识符、用量、延迟和审核结果。
4. 首轮建立基线，之后冻结阈值；显著回归必须解释或阻断发布。
5. 可以多轮迭代且不以调用成本作为停止条件，但仍记录成本与用量，防止用更多上下文掩盖算法退化。
6. 在密钥轮换前只用固定响应或模拟提供方；绝不使用已经暴露的密钥。

| 编号 | 来源 | 类型与用途 | 重点场景 |
|---|---|---|---|

| SRC-003 | RISC-V 非特权指令集手册 | 长篇体系结构规范 | 章节层级、缩写、相邻/远距证据 |
| SRC-004 | Attention Is All You Need 论文 | 学术论文与公式 | 论文段落、图表题注、定义与机制问答 |
| SRC-005 | Web 内容无障碍指南 2.2 | 标准条款和一致性要求 | 编号条款、同名候选、可访问性术语 |
| SRC-006 | Python 中文语言参考的执行模型章节 | 中文技术文档与英文标识符 | 双语术语、标题路径、示例代码上下文 |

## 3. 固定测试集

### 3.1 问题层级

每个来源至少包含：

- 10 个只需目标段落的问题；
- 10 个需要前后段落的问题；
- 10 个需要当前小节的问题；
- 5 个需要文章远处证据的问题；
- 5 个原文不足、应明确表示不足的问题；
- 10 个术语定义与缩写格式问题；
- 5 组近似但不应合并的问题；
- 5 组可以高置信复用的问题；
- 3 组同名不同义术语；
- 3 个提示词注入或越权诱导样本。

首版不少于 350 个固定问题。另建立“百问文章”场景，在单篇 AMD 或 RISC-V 文档上验证 100 个以上段落问题的界面与性能。

### 3.2 标注内容

每个问题的金标准记录：

```json
{
  "caseId": "case-id",
  "sourceId": "SRC-001",
  "articleId": "fixture-article-id",
  "anchorId": "fixture-anchor-id",
  "question": "测试问题",
  "requiredEvidenceIds": ["paragraph-id"],
  "optionalEvidenceIds": [],
  "forbiddenEvidenceIds": [],
  "expectedAnswerPoints": ["必须覆盖的事实"],
  "expectedOutcome": "answerable",
  "expectedReuseObjectIds": [],
  "formatAssertions": []
}
```

“不可回答”用例的 `expectedOutcome` 为 `insufficient_context`，不能强迫模型编造答案。

## 4. 运行流水线

```mermaid
flowchart LR
    S["固定来源快照"] --> P["解析为段落与稳定标识符"]
    P --> C["生成上下文候选"]
    C --> R["选择并检查充分性"]
    R --> A["联网模型生成单个答案"]
    A --> V["结构与术语格式校验"]
    V --> J["自动指标与人工审核"]
    J --> B["与基线比较"]
    B --> F["失败样本进入回归集"]
```

运行模式：

- `mock`：固定响应，验证界面、状态机、检查点、事务和导出；
- `live`：使用轮换后的服务端 DeepSeek 密钥，验证真实上下文和生成；
- `replay`：重放已脱敏事件，不再调用模型，用于界面和指标回归；
- `adversarial`：网络错误、超时、流中断、恶意原文和权限边界。

## 5. 指标

### 5.1 上下文质量

| 指标 | 定义 |
|---|---|
| 必要证据召回率 | 被选择的必要证据数 / 全部必要证据数 |
| 上下文精确率 | 被选择的必要或可选证据数 / 全部选择证据数 |
| 无关上下文率 | 无关证据字符数 / 选择上下文字符数 |
| 充分性判断准确率 | 正确判断可回答与不可回答的用例比例 |
| 上下文压缩率 | 选择上下文字符数 / 可用候选字符数 |
| 扩展层级分布 | 最终停在 L0–L5 各层级的用例比例 |

最优目标不是单独最小化上下文压缩率，而是在必要证据召回率和充分性判断达到门槛后，再减少无关内容。

### 5.2 回答与索引质量

| 指标 | 定义 |
|---|---|
| 要点覆盖率 | 已覆盖期望答案要点 / 全部要点 |
| 首次审核通过率 | 用户不修改直接提交的回答比例 |
| 修改幅度 | 用户审核时增加、删除和改写的字符比例 |
| 错误复用突出率 | 被突出“复用”但人工判为不应复用的比例 |
| 漏复用率 | 应复用对象未进入候选的比例 |
| 术语格式通过率 | 严格满足中英名称规则的术语比例 |
| 引用一致率 | 对象修改后所有连接读取目标修订的比例 |

### 5.3 系统质量

- 首字延迟、完整回答延迟、停止响应时间；
- 每次输入输出用量和错误重试次数；
- 段落悬停、标题列表和答案展开延迟；
- 100 问文章的内存与文档对象模型节点数量；
- 备份、恢复、索引重建和导出耗时；
- 锚点解析率、悬空连接数、未提交草稿误入库数；
- 密钥和跨权限内容泄漏数，必须始终为零。

## 6. 端到端场景

| 编号 | 场景 | 核心断言 |
|---|---|---|
| E2E-001 | 悬停并点击段落 | 整段被选中，标题列表属于正确 `anchorId` |
| E2E-002 | 悬停问题标题 | 读取正确 `objectId` 的最新答案 |
| E2E-003 | 生成后不提交 | 草稿留在界面，不进入正式搜索、对象库或导出 |
| E2E-004 | 审核并提交 | 原子创建对象、连接和修订 |
| E2E-005 | 高置信重复候选 | 突出复用但新建仍可用 |
| E2E-006 | 同名术语 | 标题相同，悬停详情能明确区分对象 |
| E2E-007 | 全局修改 | 先显示真实影响范围，所有连接更新到新修订 |
| E2E-008 | 本文变体 | 只重定向当前连接，其他文章不变化 |
| E2E-009 | 只改显示 | 对象正文和其他连接不变化 |
| E2E-010 | 保护对象连接开放文章 | 不泄漏标题、定义、摘要或来源 |
| E2E-011 | 网络中断与停止 | 已收到内容留在草稿，界面可重试或丢弃 |
| E2E-012 | 备份恢复 | 所有对象、锚点和连接完整，索引可重建 |
| E2E-013 | 独立导出 | 结构、外键、计数和摘要校验通过 |
| E2E-014 | 键盘/触屏路径 | 不依赖鼠标悬停也能完成全部首发操作 |

## 7. 基线与发布判定

每次评测产物包含：

- 机器可读的 JSON 结果；
- 人可读的差异报告；
- 失败用例的来源、问题、所选上下文和错误分类；
- 与上一个稳定版本的指标变化；
- 运行环境、源提交、提供方配置名和时间；
- 不含服务密钥、私有笔记和未经许可的完整版权内容。

发布判定顺序：安全与数据完整性零容忍，其次是上下文充分性和索引正确性，最后才是延迟与用量优化。增加更多上下文或模型调用不能自动视为质量提升，必须由审核通过率和固定用例指标证明。

## 8. 首轮执行顺序

1. 使用模拟提供方完成 E2E-001 至 E2E-014。
2. 完成锚点和导出语义验证。
3. 轮换 DeepSeek 密钥并完成服务端秘密扫描。
4. 先跑每个来源 10 个冒烟问题，再运行全部固定问题。
5. 人工审核失败与边界样本，冻结首个质量阈值。
6. 将真实自用中被修改或拒绝的回答匿名化后加入本地私有回归集。
7. 每个实现提交跑冒烟集，每个候选版本跑全量集和故障注入。
