# ReadWeave（织读）：产品规划总览

> 状态：核心闭环已实现并进入发布验收
> 规划日期：2026-07-14
> 上游基线：TriliumNext `v0.103.0` (`44f5be8`)
> 数据边界：开发与测试只使用匿名隔离数据库，不读取个人笔记内容

## 一句话定义

ReadWeave（织读，工作名）是一个“以人的真实问题为主线”的 Web-first 阅读学习器：用户在 Trilium Web 中阅读、选择段落并主动提问，系统负责最小充分上下文选择、调用联网模型、保存为稳定 ID 索引对象、提示相似内容，并让同一问答或术语定义在所有文章锚点处保持一致。

## 已确定的产品方向

1. 不再预先猜测并批量生成用户可能提出的问题。
2. 人决定“问什么、何时问、是否保存、是否复用”；AI 负责回答和减少机械操作。
3. 问答与术语定义是独立的“索引对象”：每个对象有不可变 ID，文章锚点只保存 ID 连接，不按名称绑定，也不复制答案。
4. Trilium 是笔记、关系、修订、权限继承和备份的真相源；相似度索引只是可删除、可重建的派生数据。
5. 每次生成默认是一问一答；未经用户审核的答案只停留在当前界面，不提交到索引库。
6. 先验证扩展式实现，只有扩展边界确实不足时才修改 Trilium 核心。
7. 以 Trilium Server/Web 为主，桌面端不是首发目标；所有生成必须使用联网模型。
8. 用户偏好只能显式修改，系统行为保持确定性，不从行为自动学习。

## 当前建议的产品边界

- 产品名：`ReadWeave（织读）`；开发仓库已配置，代码基于 `main` 分支持续迭代。
- 代码形态：基于 TriliumNext 的长期可合并修改版，尽量将 ReadWeave 能力隔离为独立模块。
- 首发对象：纯个人自用；用户可显式修改回答规则和显示规则，但系统不自动学习偏好。
- 首发平台：Trilium Server/Web；单机备份是 P0，同步和多人暂不考虑，移动端仅保证不损坏数据。
- 首发核心：段落 hover/点击、段落问题列表、问题 hover 展开答案、术语定义、稳定 ID 索引对象、相似标题提示、审核后提交、全局更新、备份与索引导出。
- 模型：DeepSeek API 为第一 Provider；不支持本地模型。密钥只由 Web 服务端安全配置，不进入 Trilium 数据或仓库。
- 非首发核心：自动批量出题、社交/多用户、中心化云知识库、自动替用户做学习决策。

## 当前实现

- 已在 Trilium Web 文本笔记中实现段落悬停、点击选段和持久锚点。
- 已实现问题/术语、单次联网回答、会话草稿、人工审核后保存。
- 已实现稳定对象 ID、文章锚点连接、相似标题候选、显式复用和仍可新建。

- 已实现权限继承、隐藏笔记持久化、Trilium 原生备份覆盖和可校验索引导出。
- 已加入服务器端单元/集成测试、浏览器端到端测试、构建检查和提交前隐私扫描。
