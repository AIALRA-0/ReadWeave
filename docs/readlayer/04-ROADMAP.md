# ReadWeave（织读）实施路线图

> 日期：2026-07-14
> 原则：先验证个人真实阅读动作，再冻结数据与交互；每一关都必须留下可重复的测试证据。

## 1. 发布关卡总览

| 关卡 | 目标 | 核心问题 | 退出产物 |
|---|---|---|---|
| Gate 0 | 需求与安全冻结 | 做什么、明确不做什么、秘密如何处理？ | 已确认需求、模式草案、密钥轮换 |
| Gate 1 | 无真实模型的纵向原型 | 段落悬停到审核提交的动作是否正确？ | Web 原型、锚点验证、内存提供方 |
| Gate 2 | 真实模型与上下文评测 | 最小充分上下文能否稳定产生可审核答案？ | DeepSeek 适配、评测基线、失败语料 |
| Gate 3 | 数据闭环与真实自用 | 标识符连接、全局更新、备份恢复是否可靠？ | 可日用测试版、导出、恢复证据 |
| Gate 4 | 稳定发布 | 长文、升级、权限和故障场景是否成熟？ | 候选版本、发布包、运维说明 |

不得跳过关卡。日历日期只在完成前一关的退出条件后确定。

## 2. Gate 0：需求、安全与基线

### 工作项

- 冻结 ReadWeave（织读）工作名、`readweave` 仓库名和当前 TriliumNext 基线。
- 等用户提供 GitHub 仓库地址后再配置远程仓库，不猜测或新建未经授权的远程项目。
- 吊销已经暴露的 DeepSeek 密钥，生成新密钥并仅通过服务端安全配置注入。
- 把 23 项用户决定映射到产品需求、用户体验、架构、风险和检查清单。
- 冻结首发范围：Web、单问单答、段落悬停、审核后保存、相似候选、标识符连接、备份恢复和独立导出。
- 冻结非目标：本地模型、多轮聊天、自动学习偏好、复习、测验、多人共享、旧手工块迁移和跨设备同步。
- 冻结 JSON（JavaScript 对象表示法，JavaScript Object Notation）导出模式 v1 草案。
- 建立公开测试语料清单，不使用私有日常笔记作为唯一评测依据。

### 退出条件

- PRD（产品需求文档，Product Requirements Document）中每个首发需求都有唯一标识符和验收标准。
- 用户确认产品路线；仍待确认的只有 GitHub 仓库地址和后续服务端密钥配置。
- 仓库与任何文档中不存在真实密钥。
- JSON 模式可以由标准校验器加载。

## 3. Gate 1：交互与标识符纵向原型

### 工作项

- 启动 Trilium Server/Web 的原始基线并记录测试结果。
- 实现最小 ReadWeave 客户端入口，只覆盖一篇测试文章。
- 实现段落悬停、点击整段、问题标题列表、答案悬停/聚焦和关闭。
- 使用内存提供方返回固定答案，不接真实 DeepSeek，以便先证明交互和数据边界。
- 实现浏览器草稿检查点：未审核答案留在界面但不进入 Trilium 正式数据。
- 实现 `articleId`、`anchorId`、`objectId` 和 `linkId` 的最小闭环。
- 验证锚点在编辑、拆分、合并、复制、粘贴、撤销和重做后的行为。
- 实现提交前相似候选标题列表；高置信突出“复用”，但不阻止新建。
- 实现同名候选悬停详情，显示内容、来源和标识符片段。
- 实现全局修改影响范围预览以及“全局修改/本文变体/只改显示”。

### 实验

| 编号 | 假设 | 通过证据 |
|---|---|---|
| EXP-001 | 无常驻卡片也能快速处理 100 个问题 | 真实长文任务完成时间与误操作达到阈值 |
| EXP-002 | 稳定锚点不依赖段落序号 | 编辑器变换回归全部通过或给出明确修复流程 |
| EXP-003 | 标题优先、悬停展开能降低视觉噪声 | 默认界面只显示标题，用户仍能区分同名对象 |
| EXP-004 | 审核前检查点不污染正式知识 | 重载、切换、丢弃、提交测试均保持边界 |
| EXP-005 | 原生实体映射能承载对象和连接 | 1 千、1 万、5 万连接性能曲线可接受 |
| EXP-006 | 原生权限交集不会泄漏对象内容 | 保护/开放文章交叉连接安全测试通过 |

### 退出条件

- 从段落点击到审核提交的首发闭环可运行。
- 未提交草稿不出现在对象搜索、备份承诺和导出中。
- 修改一次对象后，所有已展开连接能读取相同新修订。
- 触屏和键盘存在与悬停等价的路径。
- 没有引入多轮聊天界面或数据表。

## 4. Gate 2：DeepSeek 与最小充分上下文

### 工作项

- 在密钥轮换完成后接入 DeepSeek 服务端适配器。
- 服务地址和模型名称由配置提供，不写死到对象或代码流程。
- 实现流式显示、停止、超时、重试、错误分类、用量记录和结构化输出校验。
- 实现 L0–L5 上下文候选层级、选择、充分性检查和按需扩展。
- 用版本化 JSON 状态机记录每次上下文选择，不记录用户未授权的隐私正文。
- 首选 TypeScript 显式状态机；只有验证出复杂暂停恢复需求时才评估 LangChain/LangGraph。
- 建立精确键、全文召回和联网模型重排的相似候选流水线。
- 实现术语严格格式校验：有缩写和无缩写两种固定显示规则。
- 运行 `09-EVALUATION-HARNESS.md` 中的多来源语料和失败注入测试。

### 评测门槛冻结方式

首次完整运行只建立基线，不事先虚构阈值。基线完成后按用户真实审核样本冻结：

- 上下文充分率；
- 无关上下文占比；
- 回答一次审核通过率；
- 错误复用突出率与漏提示率；
- 术语格式通过率；
- 首字延迟、完整延迟和输入输出用量；
- 停止与错误后草稿保留正确率。

### 退出条件

- 新密钥不出现在浏览器、数据库、备份、导出、日志或版本历史。
- 每个答案都能追溯到工作流版本、模型配置、上下文标识符和提示词哈希。
- 公开语料矩阵全量运行，失败案例进入固定回归集。
- 相似候选永远不阻塞用户明确的新建或变体操作。
- 联网不可用时可查看、编辑和导出已保存对象，但生成操作明确失败且不伪造本地答案。

## 5. Gate 3：真实自用、备份恢复与导出

### 工作项

- 在日常数据库的隔离副本上升级和运行，不直接试唯一数据库。
- 对用户自行选择的多篇真实文章进行完整阅读任务，每篇允许几十到上百个问题。
- 冻结 `IndexObject`、`Anchor`、`ObjectLink` 和导出模式 v1。
- 完成对象修订、影响范围、全局修改、本文变体和只改显示。
- 完成对象删除、连接解除、孤立对象检查和派生索引重建。
- 完成 Trilium 原生备份前检查、备份、恢复、数据核对和回滚说明。
- 实现问题、答案、术语、文章、锚点和连接的独立 JSON 导出与校验报告。
- 验证保护继承：来源对象权限不因连接到其他文章而降低。
- 持续记录真实使用中的手工格式化动作，只有用户明确要求时才新增自动化规则。

### 退出条件

- 连续一个完整真实使用周期无数据丢失、静默覆盖或不可恢复锚点损坏。
- 备份恢复后对象和连接数量一致，所有外键可解析，派生索引可重建。
- 独立导出通过 JSON 模式和引用完整性校验。
- 100 问文章的阅读和编辑性能达到冻结门槛。
- 没有自动学习或暗中改变用户设置的行为。

## 6. Gate 4：稳定发布

### 工作项

- 运行全部单元、服务集成、编辑器、提供方契约、端到端、性能和安全测试。
- 对长文、重复名词、同名异义、锚点损坏、网络中断、磁盘满、备份失败和索引损坏做故障注入。
- 在数据副本上完成 Trilium 上游版本升级和回滚演练。
- 完成安装、配置、密钥轮换、备份、恢复、导出、故障排除和已知限制文档。
- 生成软件物料清单、依赖漏洞报告和许可证清单。
- 构建可重复的发布产物并记录源提交、上游标签、模式版本和评测报告。

### 发布阻断项

以下任一项未关闭即不得发布：

- 可能丢失、串改或越权展示正式对象；
- 密钥可能到达浏览器或持久化数据；
- 未提交草稿可能被当成正式知识；
- 全局修改影响范围与实际连接不一致；
- 备份不可验证或恢复无法复现；
- 导出存在悬空对象、锚点或连接；
- 固定公开语料出现未解释的显著回归；
- 无法通过键盘或触屏使用悬停功能。

## 7. 明确延期

以下内容不进入 Gate 0–4：

- 跨设备同步；
- 多人共享与协作权限；
- 本地模型；
- 多轮聊天；
- 自动学习偏好或隐式学习画像；
- 复习、测验和记忆算法；
- 知识图谱可视化；
- 旧蓝/绿手工块自动迁移。

延期不是预留半成品。只有用户重新立项、补充需求和验收标准后才进入路线图。

## 8. 每个迭代的完成定义

一个工作项只有同时满足以下条件才算完成：

1. 对应需求标识符、设计和风险已更新；
2. 正常、边界和失败路径都有自动测试；
3. 浏览器交互完成键盘、触屏、亮暗主题和高对比验证；
4. 数据变化完成备份、恢复、引用完整性和权限检查；
5. 模型变化完成固定语料回归和用量记录；
6. 文档说明真实行为和已知限制；
7. 没有真实密钥、私密笔记或敏感日志进入仓库；
8. 检查清单留下可定位的提交、报告或截图证据。
