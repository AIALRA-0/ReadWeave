# ReadWeave（织读）实现与验收状态

> 日期：2026-07-15
>
> 产品版本：0.2.0
>
> Trilium 基线：0.103.0
> 状态：个人 Web 阅读核心闭环已实现并通过隔离数据库端到端测试

## 已实现的闭环

1. 用户可按字符粒度框选任意字、词或句子。编辑器以持久范围属性保存稳定锚点；未框选时仍可点击整段建立段落锚点。
2. 未保存的空锚点没有悬停反应；已保存内容只在鼠标靠近对应文字范围时按类型高亮，移开立即恢复。右上角数字只统计问题，名词定义只在悬停预览中出现。
3. “提问/定义”是选区旁的轻量操作入口，问题、结构化名词、生成、审核、复用和编辑全部在右侧面板完成。
4. 问题保持单问单答；回答禁止环境解释、内部标签、处理过程和注释性套话。
5. 名词使用结构化字段并严格渲染为 `NPU 神经网络处理单元（Neural Processing Unit）`。
6. 卡片支持 Note、Tip、Important、Warning、Caution 五种类型，并用颜色、图标和文字共同表达。
7. 模型工作流按 6,000、12,000、30,000、80,000 字符预算逐级扩张上下文。上下文充分时只生成一次首稿；上下文已耗尽但术语可以消歧时允许同模型做一次边界化重试。检查点输出具体失败片段，后续同模型调用只返回这些片段的补丁，服务端逐字验证其余片段未被修改；自动检查仍未完全通过时仅保留真实模型草稿并列出待审核问题，不创建生产降级答案。
8. 生成结果只留在浏览器会话草稿中；只有用户明确审核保存后才创建规范对象和锚点连接。
9. 相似候选采用无卡片边框的轻量列表，显示标题相似度并突出“复用”；悬停可查看内容，用户始终能复用、新建或创建变体。
10. 规范对象按不可变 ID 连接。全局修改同步所有引用；本文变体创建新对象；只改显示仅覆盖当前连接。
11. 修改前显示连接数、文章数和可访问文章标题。
12. 独立 JSON 导出包含范围/段落选择器、原文摘录、对象、连接、工作流版本和 SHA-256 完整性摘要，已通过 JSON Schema 2020-12 校验。

## 模型设置与安全边界

- “设置 → AI / LLM → ReadWeave 模型设置”提供 API 基础地址、密码输入型 API Key、可编辑模型选择、保存、模型列表读取和连接测试。
- 设置页只返回 `hasApiKey`、密钥来源和掩码；API Key 原文不会返回浏览器，也不会进入普通设置接口。
- API Key 存在当前服务的本地非同步设置中，模型和基础地址可显式修改；应用不会从接受、拒绝或编辑行为自动学习偏好。
- 环境变量可作为未设置页面密钥时的服务端回退：`READWEAVE_DEEPSEEK_API_KEY`、`READWEAVE_API_BASE_URL`、`READWEAVE_DEEPSEEK_MODEL`。
- ReadWeave 对象、连接和导出均不包含 API Key。隐私扫描和 Git 钩子阻止常见密钥、账户、个人绝对路径进入版本库。

## 数据、权限与备份

- `_readweaveObjects` 保存规范对象，`_readweaveLinks` 保存文章锚点连接；旧版 `1.0` 数据读取时兼容并规范化为 `1.1`。
- 对象和连接保护状态继承来源文章；访问继续服从 Trilium 原生受保护会话。
- 对象与连接由 Trilium 原生备份覆盖；会话草稿只保存在 `sessionStorage`，不会自动提交或进入导出。
- 生成任务保留服务端检查点与进度；刷新或重新选择锚点后会继续轮询未完成任务。致命错误以红色精确显示；真实模型草稿若只剩自动检查问题，则以黄色待审核状态保留，所有草稿仍不会自动保存。
- 相似索引可重建，稳定对象 ID 和连接才是真相源。

## 已执行验证

- ReadWeave 服务端单元/存储测试：12 项通过，覆盖提示词规则、严格术语格式、上下文预算、复用、全局同步、变体、显示覆盖、导出和设置密钥隔离。
- 浏览器端到端：框选后的提问/定义操作浮层持续显示到外部点击或滚动、空锚点无反应、范围锚点精确高亮、问题与名词逐项悬停并移出收起、右侧生成进度、错误红框、待审核黄框、审核后保存、五类卡片、相似度、复用、三种修改和真实下载导出通过。
- 设置端到端：保存 API 基础地址、掩码密钥、模型选择与清除行为通过。
- 客户端、服务端、编辑器包和端到端项目类型检查通过；生产构建通过。
- 生成请求使用异步任务与短轮询，不受通用一分钟请求限制；网络错误按固定退避重试，页面可见每次检查、上下文扩张和定点修复。
- 真实 DeepSeek 验收发现并修复了把产品名 WARP 杜撰为英文全称的问题；最终答案使用“应急网络服务（WARP）”保留原文指向且不补不存在的词源，草稿未保存。

## 使用者验收

实现完成不等于阅读习惯已经定型。产品所有者应在日常数据库副本中用至少三篇不同类型文章验证：锚点粒度、问题密度、名词冲突、答案直接性、模型选择和悬停节奏。偏好变化只进入显式设置或新需求，不启用隐式行为学习。
