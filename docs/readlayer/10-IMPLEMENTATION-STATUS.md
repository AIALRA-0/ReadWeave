# ReadWeave（织读）实现与验收状态

> 日期：2026-07-14
> 产品版本：0.1.0
> Trilium 基线：0.103.0
> 状态：核心个人 Web 阅读闭环已实现，等待真实密钥轮换与日常数据库副本恢复演练

## 已实现的端到端闭环

1. 在 Trilium Web 文本笔记内悬停段落并点击，编辑器把 `data-readweave-anchor-id` 保存为稳定锚点。
2. 右侧 ReadWeave 面板只显示该锚点的问题/术语标题；悬停或键盘聚焦后展开正文。
3. 用户输入一个问题或术语，系统按确定性预算选择最小充分上下文，并由服务端调用 DeepSeek。
4. 生成结果先保存在当前浏览器的会话草稿中；只有点击“已审核并保存”才写入知识库。
5. 保存时创建规范对象和锚点连接；相似候选只突出复用，不阻止新建或变体。
6. 规范对象按不可变 Trilium 笔记 ID 连接。全局修改会实时影响所有引用；本文变体和只改显示保持独立语义。
7. 修改前显示连接数、文章数和可访问文章标题。
8. 当前文章可导出文章、段落锚点、规范对象和连接，并包含 SHA-256（安全散列算法 256 位，Secure Hash Algorithm 256-bit）完整性摘要。

## 数据、权限与备份

- `_readweaveObjects` 保存规范对象，`_readweaveLinks` 保存文章锚点连接；两者位于 Trilium 隐藏子树。
- 对象和连接的保护状态继承来源文章；读取和导出均经过 Trilium 当前受保护会话的内容可用性检查。
- 规范对象与连接属于 Trilium 数据库，因此由原生每日、每周、每月备份统一覆盖。
- 相似度候选由可重建的确定性标题归一化与二元字符相似度生成，不是连接真相源。
- 草稿只保存在浏览器 `sessionStorage`，不会自动提交，也不会进入索引导出。

## 模型配置

只在 Web 服务端环境中配置：

```text
READWEAVE_DEEPSEEK_API_KEY=<rotated-server-only-key>
READWEAVE_DEEPSEEK_MODEL=deepseek-chat
```

固定调用官方 DeepSeek Chat Completions 接口；浏览器、笔记、导出、日志和仓库中均不保存密钥。测试模型只允许在 `TRILIUM_INTEGRATION_TEST=memory` 的匿名内存数据库中启用。
