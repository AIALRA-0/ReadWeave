# ReadWeave（织读）

ReadWeave 是基于 TriliumNext `v0.103.0` 的个人 Web 阅读器修改版。用户在原文中精确框选文字，问题与名词定义在右侧完成；正文只保留虚线锚点和问题数量徽标，不插入占用阅读空间的问答卡片。

## 使用方法

1. Windows 双击仓库根目录的 `Start-ReadWeave.cmd`；它会构建最新版、使用独立数据目录后台启动，并自动打开浏览器。需要停止时双击 `Stop-ReadWeave.cmd`。
2. 首次启动按页面提示创建这套独立 ReadWeave 数据库；之后打开“设置 → AI / LLM → ReadWeave 模型设置”。
3. 填入 OpenAI 兼容服务的 API 基础地址和 API Key，输入或从测试结果中选择模型，然后保存并测试连接。
4. 打开文本笔记并框选任意字、词或句子；选区显示虚线高亮和“提问/定义”操作，具体编辑始终在右侧面板进行。
5. 一次只写一个问题。点击生成后，结果只作为当前浏览器会话草稿显示；审核或修改满意后点击“已审核并保存”。
6. 正文锚点右上角数字只表示问题数量。悬停锚点才显示问题标题和名词定义；悬停问题标题会自然展开答案。
7. 名词严格使用 `NPU 神经网络处理单元（Neural Processing Unit）` 格式。缩写、中文全称和英文全称分别编辑和保存。
8. 相似候选会突出“复用”，但不会阻止新建。修改已有对象前会显示影响范围，并提供“全局修改/本文变体/只改显示”。
9. “导出本文问题—锚点索引”会生成独立 JSON 文件，连接依赖稳定标识符而不是名称。

段落点击仍保留为未框选文本时的整段锚点入口，但推荐使用精确框选。

## 模型与失败规则

DeepSeek 是默认服务和模型，但基础地址、模型名都可在设置中替换。生成器从选区、相邻内容、章节到全文逐级扩张上下文；信息不足或质量检查失败时，会继续扩张、修复并重新验证。它不会用本地模型、抽取句、模板或假答案降级。所有尝试仍失败时，界面明确报错且不产生可保存的伪答案。

设置页 API Key 保存在当前 Web 服务的本地、非同步设置中；浏览器只会收到掩码，不会收到原文。也可以使用环境变量配置，设置页中的密钥优先：

```text
READWEAVE_DEEPSEEK_API_KEY=<server-only-key>
READWEAVE_API_BASE_URL=https://api.deepseek.com
READWEAVE_DEEPSEEK_MODEL=deepseek-chat
```

不要把真实值写入仓库文件、笔记、导出、日志、截图或聊天记录。仓库中的示例和自动测试只允许使用明显的假值。
