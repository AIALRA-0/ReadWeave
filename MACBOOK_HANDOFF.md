# ReadWeave（织读）MacBook 开发交接

> 交接快照：2026-07-18
> 上游基线：TriliumNext 0.103.0
> 产品形态：单用户、Web 优先、联网模型驱动的高度定制阅读器

## 1. 初衷

ReadWeave 不是一个“替用户自动学习”的通用 AI 应用。它从用户真实的精读习惯出发：阅读原文、精确选中某个字词或句子、提出问题或记录术语，再把复制、粘贴、排版、查重、索引和同步这些机械步骤自动化。

核心判断是：学习器必须先贴合个人学习习惯，再自动化其中可被可靠自动化的部分；行为必须确定、可审核、可追溯，不能暗中学习偏好，也不能用低质量回退伪装成功。

## 2. 成熟产品目标

- 以 Trilium Notes 的笔记树、编辑器、权限继承和备份为基础，主场景是 Web 端；
- 用户按字符粒度框选原文，提问和定义的编辑、生成、审核均在右侧进行，不挤占正文阅读空间；
- 正文只显示轻量虚线锚点与问题数量；没有内容的文字不响应；
- 问题保持一问一答；模型结果只作为会话草稿，用户明确审核后才保存；
- 术语与问答是由稳定标识符连接的规范对象，可跨文章复用并全局同步；
- 相似内容提示“复用”，但不阻止新建或创建变体；
- 所有模型调用必须联网，API 服务、密钥与模型由用户显式设置；
- 任何错误都必须暴露并继续定点修复，禁止本地模型、模板答案、抽取句或静默降级。

## 3. 已确定的交互契约

### 正文和锚点

- 精确框选任意字、词或句子；选区旁只出现轻量“提问/定义”操作入口；
- 保存后的锚点常态仅有向下留出少量距离的彩色虚线，不使用背景色；
- 鼠标进入锚点立即高亮对应范围，移出立即完全清除；不能要求点击，也不能有白色虚线残留；
- 纯问题锚点只高亮，不弹框；术语锚点悬停显示定义预览；
- 问题数量只显示简洁数字，不带圆圈；名词定义不计入数字；
- 外层 Note 与内层 Tip 可以重叠：外层保留蓝色，内层术语显示绿色；悬停术语时只突出术语范围并显示术语框；
- 没有保存内容的范围不应有悬停响应。

### 右侧面板

- Ask 默认为 Note，Define 默认为 Tip；用户可改为 Note、Tip、Important、Warning、Caution 之一；
- 五类未选择时都为灰色，只有被选中的一类呈对应颜色；不加花哨外框；图标与文字视觉居中；
- “问题/术语”等普通操作按钮不使用五色彩虹式配色；
- 问题优化是显式勾选项：不丢失任何信息密度和准确性，只整理表达；
- 提问/定义切换应随悬停自然切换或消失，不能黏住；
- 相似候选采用自然轻量列表，显示相似度；用户可悬停看内容并选择复用、新建或变体；
- 修改规范对象前先显示影响范围，再选择“全局修改 / 本文变体 / 只改显示”；默认语义是全局同步。

### 内容格式

- 术语严格使用 `NPU 神经网络处理单元（Neural Processing Unit）`，不是 `NPU（神经网络处理单元，Neural Processing Unit）`；
- abbreviation、中文全称、英文全称均可为空；默认由模型生成，用户输入优先；
- 回答直接回答问题，不加入“根据上下文”“原文指出”等环境解释、过程说明或注释；
- 中文回答使用分号组织，不残留中文句号；
- 问题与回答默认一问一答，不做聊天记录；
- 导出支持独立的“问题 + 锚点连接索引”JSON，并以稳定 ID 而不是名称连接。

## 4. 回答质量契约

问题回答采用证据驱动的八维闭环：

1. 定义与命名；
2. 底层构造；
3. 层次关系；
4. 参数配置；
5. 行为语义；
6. 测试判据；
7. 数字推导；
8. 实现选择与证据闭环。

并非每个问题都有足够证据填满全部维度。模型不能为了形式完整而发明信息；证据不足时应扩大上下文或明确边界。当前实现先生成证据清单，再要求严格 JSON 八段输出；检查器定位失败片段，后续调用只修补指定片段，服务端逐字验证其他片段未变化。最多进行有限轮定点修复；仍不合格则明确报错且不可保存。

特别规则：

- 覆盖问题中的每个子问、因果约束、备选项、数字和失败证据；
- 英文术语或产品名必须来自原文证据，并按规范格式传播；
- 允许受证据支持的语义改写，不要求逐字复述原文；
- 禁止凭空添加命令、网址、界面状态、端口行为或工具；
- 禁止在缺少起点、并行/串行关系时擅自计算总耗时；
- 草稿永不自动保存。

## 5. 数据与权限

- `_readweaveObjects`：问答/术语规范对象；
- `_readweaveLinks`：文章、范围锚点与对象之间的连接；
- 真相源是不可变对象 ID 与连接，名称只用于显示与相似检索；
- 全局修改更新所有引用；本文变体创建新对象；只改显示仅覆盖当前连接；
- 访问权限与保护状态继承 Trilium 原生文章；
- 对象与连接进入 Trilium 原生备份；会话草稿只在 `sessionStorage`；
- 当前不做多人共享、复习和测验；允许导出为以后共享或图谱使用；
- 当前不做跨设备同步，但必须维持可靠备份边界。

## 7. MacBook 首次启动

要求：macOS、Git、Xcode Command Line Tools、Node.js 24.15.0。Apple Silicon 与 Intel 均应重新安装依赖，绝不能复制 Windows 的 `node_modules`。

```bash
cd ReadWeave
bash scripts/readweave/bootstrap-macos.sh
bash scripts/readweave/start-macos.sh
```

打开 `http://127.0.0.1:8082`，首次启动建立独立数据库，然后在“设置 → AI / LLM → ReadWeave 模型设置”中填写 API 基础地址、API Key 和模型。不要把真实密钥写进 `.env`、源码、文档、测试、截图或 Git。

停止：

```bash
bash scripts/readweave/stop-macos.sh
```

如需只启动开发监听模式：

```bash
pnpm server:start
```

这会使用上游默认的 `apps/server/data`，与正式一键启动使用的 `apps/server/data-readweave` 不同；涉及真实笔记前先确认数据目录。

## 8. 开发验证命令

```bash
pnpm readweave:privacy
pnpm --dir apps/server test -- src/services/readweave_ai.spec.ts src/services/readweave_settings.spec.ts src/services/readweave_repository.spec.ts src/services/readweave_engine.spec.ts
pnpm --dir apps/server build
pnpm --dir apps/server-e2e exec playwright test src/readweave.spec.ts --project=chromium
```

实时模型测试会产生 API 费用，只有在本机安全配置密钥后运行：

```bash
pnpm --dir apps/server test -- src/services/readweave_ai.live.spec.ts
```

每次改动必须至少验证直接相关测试；锚点、悬停、选择范围和右侧面板改动必须跑浏览器端到端；AI 提示词、检查点和修补协议改动必须跑单元测试与真实模型回归。

## 9. 安全红线

- 不上传 API Key、密码、账户、邮箱、令牌、Cookie、个人绝对路径、真实用户笔记或数据库；
- 不打包 `apps/*/data*`、`.env*`、`*.db`、`*.sqlite*`、日志、截图、测试产物和浏览器会话；
- 不将用户曾在聊天中给出的任何秘密复制进代码或 Git 历史；
- 发布前运行 `pnpm readweave:privacy` 并人工检查 `git diff`；
- 不把当前工作目录替换成上游干净版本，不用破坏性 Git 命令清除未提交实现。

## 10. 下一阶段工作方式

继续以真实文章和真实提问习惯做端到端迭代，而不是扩张不需要的功能。优先级依次为：阅读交互稳定性、回答证据与直接性、数据一致性、失败可恢复性、性能、视觉微调。发现生成错误时先加入可复现测试，再修提示词/检查点/确定性校验；发现交互错误时先覆盖真实鼠标 hover、精确框选、嵌套范围和移出清理。
