# 给另一台 Codex 的初始提示词

把下面整段作为新任务的第一条消息发送给 Codex：

---
你现在接手 ReadWeave（织读）项目。它是基于 TriliumNext 0.103.0 的个人 Web 精读器，不是通用自动学习产品。请先完整阅读仓库根目录的 `MACBOOK_HANDOFF.md`、`README_READWEAVE.md`，以及 `docs/readlayer/` 下的 PRD、UX、架构、追溯清单、评估 harness 和实现状态，然后检查实际代码与 Git 工作区；文档是意图，测试和当前代码是实现证据，出现不一致时先报告并补齐可复现测试。

项目的根本目标是贴合用户真实精读习惯：用户在原文中按字符粒度框选，问题或术语在右侧生成、编辑和审核；正文只保留轻量虚线锚点和问题数字，术语才有悬停预览；模型草稿只有用户审核后保存。规范对象通过稳定 ID 跨文章复用与全局同步，相似内容提示复用但不禁止新建或变体。

必须遵守：Web 优先；联网模型；API 服务、密钥和模型由用户显式配置；不自动学习偏好；不做本地模型或模板回退；失败必须显示并定点修复；禁止为了八维格式发明信息；术语严格写成 `NPU 神经网络处理单元（Neural Processing Unit）`；回答直接、无环境套话，中文用分号；问答默认一问一答；权限和保护继承 Trilium；不自动保存草稿。

开发时保持当前未提交工作态，不得用上游代码覆盖，不得使用 `git reset --hard` 或类似破坏性命令。严禁把 API Key、密码、账户、个人路径、真实笔记、数据库、日志或会话上传到 GitHub。不要读取或打印秘密；真实模型测试只从本地安全设置获得密钥。发布前运行隐私扫描并检查变更。

先执行以下接管动作：

1. 确认 Node.js 24.15.0 与 pnpm 10.33.4；执行 `bash scripts/readweave/bootstrap-macos.sh`；
2. 阅读 `git status`，保留所有现有改动；
3. 运行服务端 ReadWeave 非实时测试、生产构建和隐私扫描；
4. 用 `bash scripts/readweave/start-macos.sh` 启动隔离数据目录；
5. 用 Chromium 真实验证精确框选、Note 外层 / Tip 内层、问题数字、纯问题无弹框、术语悬停预览、移出完全清理、右侧生成和审核保存；
6. 若本机已安全设置模型，再运行真实模型测试；不得因没有密钥而写入假密钥或改成降级答案；
7. 给我一份简洁接管报告：环境、测试结果、实际阻塞、下一项最值得做的端到端改进。未经我提出，不扩张到多人协作、复习、测验或隐式学习画像。

后续每次收到反馈，都要对齐到最小准确颗粒度，先复现、加入回归测试、实现、端到端验证，再交付可直接使用的结果。答案质量必须由证据闭环衡量，不能仅凭看起来通顺。
